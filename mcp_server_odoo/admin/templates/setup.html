How access control works

You have two layers of control over what your AI can do in Odoo:

1 Odoo API key permissions

Your API key inherits the permissions of your Odoo user account. The AI can only access the same models, records, and actions that you can access when you log in to Odoo directly. Want to limit access? Create a dedicated Odoo user with restricted permissions and use that user's API key.

2 Per-tool approval in your AI client

Your AI client (e.g. Claude) lets you choose per MCP tool whether to always allow it, require your approval each time, or block it entirely. For example, you can allow search_records automatically but require approval for create_record or delete_record.

  • Your API key is encrypted at rest and only used server-side to authenticate requests to your Odoo instance.
  • You can revoke your API key in Odoo at any time, instantly cutting off access. You can also remove your connection above.
